# Order Module — Laravel

Laravel-модуль: адаптивный блок услуг + форма заявки с авторизованным просмотром заявок.

## Что сделано

- Главная страница (`/`) — блок услуг (3 карточки) + форма заявки
- Валидация формы: backend (Laravel `validate`) + frontend (HTML-атрибуты `required`, `maxlength`, `minlength`)
- Сохранение заявки в SQLite через Eloquent ORM
- Flash-сообщение об успешной отправке
- Страница списка заявок (`/orders`) — **только для авторизованных пользователей** (middleware `auth`)
- Страница входа (`/login`) с защитой от перебора через стандартный `Auth::attempt`
- Кнопки «Войти» / «Выйти» в шапке в зависимости от статуса сессии
- Адаптивная вёрстка на чистом CSS (mobile + desktop)

## Стек

- PHP 8.3, Laravel 13
- Blade-шаблоны
- SQLite (по умолчанию, без доп. настроек)
- Чистый CSS (без фреймворков)

## Структура

```text
app/
  Http/Controllers/
    OrderController.php          # index, store, list
    Auth/LoginController.php     # showLoginForm, login, logout
  Models/Order.php               # fillable: name, email, message
database/
  migrations/..._create_orders_table.php
  seeders/DatabaseSeeder.php     # создаёт admin из ADMIN_EMAIL/ADMIN_PASSWORD в .env
public/
  css/app.css                    # общие стили (кнопки, форма) — подключается в layout
resources/views/
  layouts/app.blade.php          # общий layout с навигацией
  home.blade.php                 # услуги + форма
  orders.blade.php               # список заявок (только auth, пагинация по 20)
  auth/login.blade.php           # форма входа
routes/web.php                   # GET /, POST /orders (throttle:10,1), GET /orders (auth),
                                 # /login, POST /login (throttle:5,1), /logout
```

## Запуск

```bash
# 1. Клонировать репозиторий
git clone <url>
cd order_module

# 2. Установить зависимости
composer install

# 3. Создать .env и сгенерировать ключ
cp .env.example .env
php artisan key:generate

# 4. Запустить миграции (SQLite создаётся автоматически)
php artisan migrate

# 5. Задать учётные данные администратора
# В .env установить:
# ADMIN_EMAIL=admin@example.com
# ADMIN_PASSWORD=your_secure_password

php artisan db:seed

# 6. Запустить dev-сервер
php artisan serve
```

Открыть в браузере: <http://localhost:8000>

## Валидация

### Backend (`OrderController@store`)

- `name` — обязательное, строка, макс. 255 символов
- `email` — обязательное, корректный email, макс. 255 символов
- `message` — обязательное, строка, мин. 10, макс. 2000 символов (ограничение применяется и на сервере, и через `minlength`/`maxlength` в HTML)

### Frontend (HTML5-атрибуты)

- Все поля имеют `required` — браузер блокирует отправку пустой формы
- `maxlength="255"` на полях имя и email
- `minlength="10"` и `maxlength="2000"` на текстовом поле

Двойной слой защиты: браузерная валидация как UX-подсказка, серверная — как гарантия корректности данных.

Ошибки серверной валидации отображаются под каждым полем, значения сохраняются через `old()`.

## Архитектурные решения

**SQLite вместо MySQL/PostgreSQL** — нет внешних зависимостей; база создаётся автоматически при первом `migrate`. Подходит для демонстрационного проекта и упрощает развёртывание. Переключение на другую СУБД — изменение одной строки в `.env`.

**Blade без JS-фреймворков** — задача не требует SPA-поведения; серверный рендеринг упрощает отладку и исключает проблемы гидратации. Flash-сообщения реализованы через сессию Laravel.

**Middleware `auth` на `/orders`** — стандартный механизм Laravel; не требует написания кастомного кода. При попытке анонимного доступа фреймворк сам редиректит на `/login` (задано через `$redirectTo` в `Authenticate` middleware).

**`firstOrCreate` в сидере** — повторный запуск `db:seed` не создаёт дубликаты администратора. Учётные данные берутся из `ADMIN_EMAIL` / `ADMIN_PASSWORD` в `.env` — никаких паролей в коде.

**Общий CSS в `public/css/app.css`** — стили кнопок и форм вынесены в единый файл, подключаемый через layout; страничные `<style>`-блоки содержат только локальные переопределения.

**Чистый CSS без препроцессоров** — задание не оговаривало инструментарий; серверный рендеринг упрощает отладку без лишних зависимостей.

## Оптимизации при масштабировании

| Задача | Решение |
| --- | --- |
| Рост таблицы `orders` | Пагинация реализована (`paginate(20)`); для больших объёмов — индекс на `created_at` |
| Защита от спама и брутфорса | Реализована: `throttle:10,1` на форму заявки, `throttle:5,1` на логин |
| Email-уведомления | Laravel Mailables + очередь (`Queue`) |
| Несколько администраторов | Поле `role` в `users` или пакет `spatie/laravel-permission` |
| Смена СУБД | Изменить `DB_CONNECTION` в `.env`, перезапустить миграции |
