# Order Module — Laravel

Laravel-модуль: адаптивный блок услуг + форма заявки.

## Что сделано

- Главная страница (`/`) — блок услуг (3 карточки) + форма заявки
- Backend-валидация формы (имя, email, сообщение ≥ 10 символов)
- Сохранение заявки в SQLite через Eloquent ORM
- Flash-сообщение об успешной отправке
- Страница списка заявок (`/orders`) с таблицей
- Адаптивная вёрстка на чистом CSS (mobile + desktop)

## Стек

- PHP 8.3, Laravel 13
- Blade-шаблоны
- SQLite (по умолчанию, без доп. настроек)
- Чистый CSS (без фреймворков)

## Структура

```text
app/
  Http/Controllers/OrderController.php  # index, store, list
  Models/Order.php                       # fillable: name, email, message
database/
  migrations/..._create_orders_table.php
resources/views/
  layouts/app.blade.php                  # общий layout
  home.blade.php                         # услуги + форма
  orders.blade.php                       # список заявок
routes/web.php                           # GET /, POST /orders, GET /orders
```

## Запуск

```bash
# 1. Клонировать репозиторий
git clone <url>
cd order_module

# 2. Установить зависимости
composer install

# 3. Создать .env и сгенерировать ключ
cp .env.example .env
php artisan key:generate

# 4. Запустить миграции (SQLite создаётся автоматически)
php artisan migrate

# 5. Запустить dev-сервер
php artisan serve
```

Открыть в браузере: <http://localhost:8000>

## Маршруты

| Метод | URL       | Действие                         |
|-------|-----------|----------------------------------|
| GET   | /         | Главная: услуги + форма заявки   |
| POST  | /orders   | Сохранение заявки (валидация)    |
| GET   | /orders   | Список всех заявок               |

## Валидация

- `name` — обязательное, строка, макс. 255 символов
- `email` — обязательное, корректный email, макс. 255 символов
- `message` — обязательное, строка, мин. 10 символов

Ошибки валидации отображаются под каждым полем, значения полей сохраняются через `old()`.

## Что можно улучшить при наличии времени

1. **Пагинация** списка заявок
2. **Email-уведомление** менеджеру при новой заявке (Laravel Mailables)
3. **Сортировка и фильтрация** заявок по дате, имени, статусу
4. **Статусы заявок** (новая / в работе / завершена) + возможность обновления
5. **Авторизация** для просмотра списка заявок (только для администратора)
6. **Honeypot или rate limiting** для защиты формы от спама
7. **Тесты** (Feature-тесты формы и валидации через PHPUnit)
8. **Переменные среды** для имени компании, описания услуг (вынести из шаблона)
